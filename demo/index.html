Scrollify.

    About

    Scrollify. Do things on scroll.

    What sorts of things?

    Well, whatever you want. Common effects such as parallax or toggling a class on/off are possible and bundled within. However, you may also easily create your own custom scroll function, and pass it in to Scrollify. The built-in effects include:

    • Stick: sticks an element to a particular point and holds in there for a pre-defined number of pixels.
    • Parallax: move stuff on scroll. A subtle effect that transistions elements more slowly (or perhaps quickly) than the speed of the user-scroll.
    • Toggle: turn a CSS class on or off if the element crosses a particular threshold on the page.
    • Translate: translate an element along the X- or Y- axis.
    • Rotate: rotate an element via scroll.
    • Scale: scale an element.
    • Fade: change an element's opacity.

    Robust

    The effects work perfectly fine when scrolling in reverse as well, and after resizing the viewport, too!

    Notes

    Scrollify works by first calculating an element's position in the page so that it may be manipulated on scroll. It is important to note that as the page loads, this position may jump around as the DOM is constructed and images are loaded, etc. Therefore, it is important that Scrollify'd elements are not initialized until the page has finished loading all images and the DOM is stable.

    A note on implementation: Scrollify uses matrix transformations in its calculations -- in this way multiple element transforms may be stacked and animated on a single element (it is also a lot more efficient and speedy).

    If you're trying to Scrollify an element whose position on the page is dependant on assets loading above of it, you may wish to use window.addEventListener('load', ...) rather than the more common window.addEventListener('DOMContentLoaded', ...).

    Getting Started

    Add a Scene and Effect

    The easiest way to get started is to create a new Scrollify instance, with the DOM element you'd like to manipulate. Next, create a "Scene", and define which effect to apply.

    By default, the Scene will be active for the duration of the element's time in the viewport. That is to say: the Scene starts as soon as the element enters the viewport, and ends when it leaves. 

    
              
							NOTE: Ideal API :
              new Scrollify('#transformer') // the element to scrollify
                .addScene({
                  'effects': [{
                    'scale': {              // what effect to apply
                      'to': 1.5,            // some options for the effect
                      'from': 1.0
                    }
                  }]
                });
    Result:

    This element will scale from 1 to 1.5x as it moves across the viewport.

    Define Start and Duration

    If you wish to have a Scene start and stop at a particular place, it's easy. "Start" is a value between 0 and 1 that corresponds to a position between the bottom of the viewport (0) and the top (1). "Duration" is a value in pixels. Both are optional arguments; if either is not present, the Scene will fallback on its default start / stop values. 

    
		           	// TODO: FIX up start units, add stop, incorporate duration units
								new Scrollify('#transformer-2')
									.addScene({
										'start': 0.2,			// start when the element reaches 20% of the viewport
										'duration': 300,	// do something for 300 pixels
										'effects': [{
											'scale' : {
												'to': 1.5,
												'from': 1.0
											}
										}]
									});

    Result:

    This element will only start transforming at 20% of the way up from the bottom, and stop after 300px.

    Custom Easing

    Use a custom (built in) easing function. The easing function is specific to the Scene it is defined on, and applies to all child effects. 

    
                 // TODO: FIX up start units, add stop, incorporate duration units
                new Scrollify('#transformer-3')
                  .addScene({
                    'start': 0.2,
                    'duration': 300,
                    'easing': easeInElastic,  // use a built in easing function
                    'effects': [{
                      'scale' : {
                        'to': 1.5,
                        'from': 1.0
                      }
                    }]
                  });

    Result:

    This element will only start transforming at 20% and stop after 300px. It uses an elastic easing function.

    Add Multiple Effects

    You can also specify multiple effects within the same scene:

    
							///							NOTE: Ideal API :
              new Scrollify('#transformer-4')
                .addScene({
                  'start': 0.5,
                  'duration': 100,
                  'effects': [{
                    'scale': {
                      to: 1.5,
                      from: 1.0
                    }, {
                      'rotate': {            // add other effects if you wish
                        rad: Math.Pi / 2
                      }
                    }
                  }]
                });

    This element will scale and rotate after it reaches 50%, for 100px.

    Chain Scenes

    You can also chain multiple scenes together for more sophisticated and complex interactions. Note, as there is no duration, the Scene will end when the element crests the top of the viewport. 

    
							new Scrollify('#multiple')
								.addScene({
									'start': 0.2,
									'easing': easeIn,
									'effects': [{
										'name': translateY,
										'options': {
											'to': 0,
											'from': 400
										}
									}]
								})
								.addScene({
									'trigger': '#otherThing', // another element can be used to start / stop the scene
									'start': 0.5,             // now, when "otherThing" is at 50%, will the scene start
									'applyTransform': false,  // specifies that the effect is not a css transform. TODO automate this
									'easing': oscillate,
									'effects': [{
										'name': fade,
										'options': {
											'to': 0
										}
									}]
								});

    This element will translate at 50%, and then blink when #otherThing reaches 50%.

    I'm #otherThing, the trigger for the above paragraph.

    Use an Implied Scene

    You can also just apply an effect directly, and Scrollify will set you up with some default scene options (ie start, duration etc). Here, we're passing in an HTML element directly:

    
							var element = document.querySelector('.scrollify');
							var options = {
								range: 300
							};

							new Scrollify(element).addEffect(parallax, options);
    Stick

    There is a sticky function as well; keep in mind that you'll need to CSS a few things, though. It's probably easiest if you wrap your element in an empty "spacer" element, which would serve as a reference point for changes in the display property of the sticky element and prevent the page from collapsing when it's taken out of the flow.

    
							<div class="ref"><!-- the reference element -->
								<img class="scrollify" src="photo.jpg"><!-- target element -->
							</div>
						
    
              new Scrollify('#stick')                    // the element to target
                .addScene({
                  'start': 0.5,                          // 'start': when at 50% of the viewport
                  'duration': 300,                       // 'duration': 300 pixels
                  'effects': [{
                    'name': stick                        // the effect to apply within the progress of the scene
                  }]
                });

    Custom Effects

    You can also roll your own custom effect. Have a look in the effect.js file to glean further insight.

    
							Scrollify.fx.skew = ...

							new Scrollify('#custom')
								.addEffect(function(progress) {
									var degrees = 90 * progress;
									this.element.style.transform = 'skew('+degrees+'deg)';
								});
